## 模块化组织原则

### 按功能模块划分

```
src/
├── user/               # 用户模块
│   ├── api.ts          # 用户API
│   ├── types.ts        # 用户类型
│   ├── store.ts        # 用户状态
│   └── hooks/          # 用户组合式函数
├── order/              # 订单模块
│   ├── api.ts
│   ├── types.ts
│   └── store.ts
└── product/            # 产品模块
    ├── api.ts
    └── types.ts
```

### 公共资源集中管理

```
src/
├── api/                # 所有API（集中管理）
├── components/         # 所有公共组件
├── hooks/              # 所有组合式函数
└── utils/              # 所有工具函数
```

### 分层架构

```
src/
├── pages/              # 视图层：页面组件
├── components/         # 视图层：公共组件
├── hooks/              # 逻辑层：可复用逻辑
├── stores/             # 数据层：状态管理
├── api/                # 数据层：API调用
└── utils/              # 工具层：通用工具
```

### 模块划分判断标准

| 维度 | 按功能模块划分 | 按类型分层 |
|------|----------------|------------|
| **业务复杂度** | 复杂、完整功能（CRUD、独立状态） | 简单、通用功能 |
| **复用范围** | 单一业务场景使用 | 被多个功能模块复用 |
| **团队分工** | 按业务线分工 | 按技术栈分工 |
| **独立性** | 可能独立拆分为子应用 | 基础设施，不会独立拆分 |
| **典型示例** | user、order、payment、cart | api、utils、components、hooks |

**判断流程：**
1. 是否包含 3 个或以上不同类型的文件（API、Store、Hooks、Pages）？是 → 功能模块
2. 是否仅被单一业务场景使用？是 → 功能模块
3. 是否有明确的责任人/团队？是 → 功能模块
4. 未来是否可能独立出去？是 → 功能模块

**推荐方案：混合模式**
- 基础设施采用类型分层（api、utils、components、hooks、stores）
- 复杂业务模块采用功能模块划分（user、order、payment等）

### static 目录

- 用于存放静态资源（图片、字体等）
- 直接复制到输出目录，不经过打包处理
- 适合存放不会改变的资源

### uni_modules 目录

- 用于存放 uni-app 插件
- 支持从插件市场导入
- 插件示例：
  ```
  uni_modules/
  └── uni-ui/
      └── components/
          └── uni-badge/
  ```

## 环境配置

### 环境变量文件

```bash
# .env（所有环境）
VITE_APP_TITLE=我的应用
VITE_APP_VERSION=1.0.0

# .env.development（开发环境）
VITE_APP_API_BASE_URL=http://localhost:3000/api

# .env.production（生产环境）
VITE_APP_API_BASE_URL=https://api.example.com
```

### 环境变量使用

```typescript
// ✅ 推荐（Vite要求VITE_前缀）
const apiBaseUrl = import.meta.env.VITE_APP_API_BASE_URL;

// 类型声明
// env.d.ts
interface ImportMetaEnv {
  readonly VITE_APP_TITLE: string;
  readonly VITE_APP_VERSION: string;
  readonly VITE_APP_API_BASE_URL: string;
}
```

## 路径别名配置

### vite.config.ts 配置

```typescript
import { defineConfig } from "vite";
import vue from "@vitejs/plugin-vue";
import { resolve } from "path";

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      "@": resolve(__dirname, "src"),
      "@components": resolve(__dirname, "src/components"),
      "@api": resolve(__dirname, "src/api"),
      "@utils": resolve(__dirname, "src/utils"),
      "@stores": resolve(__dirname, "src/stores"),
      "@types": resolve(__dirname, "src/types"),
      "@hooks": resolve(__dirname, "src/hooks"),
    },
  },
});
```

### tsconfig.json 配置

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],
      "@components/*": ["src/components/*"],
      "@api/*": ["src/api/*"],
      "@utils/*": ["src/utils/*"],
      "@stores/*": ["src/stores/*"],
      "@types/*": ["src/types/*"],
      "@hooks/*": ["src/hooks/*"]
    }
  }
}
```
