# 项目概述

### 核心技术

| 技术 | 版本 | 说明 |
|------|------|------|

## 项目定位

本项目是一个跨端应用，支持微信小程序、支付宝小程序、H5等多平台运行，采用**一套代码多端运行**的开发模式。

## 架构设计原则

### 1. 组件化原则

- **原子组件**：不可再分的基础UI组件（如按钮、输入框）
- **分子组件**：由原子组件组合成的复合组件
- **组织组件**：由分子组件和原子组件组成的复杂组件
- **模板组件**：页面级组件，组合上述所有组件

### 2. 模块化原则

- 按功能模块划分代码结构
- 模块间低耦合、高内聚
- 依赖关系单向流动

### 3. 响应式原则

- 适配多种屏幕尺寸
- 支持横竖屏切换
- 适配不同平台的API差异

### 4. 类型安全原则

- 优先使用TypeScript类型
- 避免使用`any`类型
- 充分利用类型推导

## 跨端设计原则

### 1. 条件编译

使用uni-app条件编译进行平台差异处理：

```typescript
// #ifdef H5
// H5平台特有代码
// #endif

// #ifdef MP-WEIXIN
// 微信小程序特有代码
// #endif

// #ifndef MP
// 非小程序平台
// #endif
```

### 2. 平台适配

```typescript
import { getSystemInfoSync } from 'uni-app'

const platform = getSystemInfoSync().platform // 'ios' | 'android' | 'devtools'
```

### 3. 组件库适配

```vue
<script setup lang="ts">
// #ifdef H5
import { Button as VanButton } from 'vant'
// #endif

// #ifdef MP
import TnButton from '@tuniao/tnui-vue3-uniapp/components/button/src/button.vue'
// #endif
</script>

<template>
  <!-- #ifdef H5 -->
  <van-button type="primary">按钮</van-button>
  <!-- #endif -->

  <!-- #ifdef MP -->
  <tn-button type="primary">按钮</tn-button>
  <!-- #endif -->
</template>
```

### 4. API封装层

封装跨端统一API，内部处理平台差异：

```typescript
// utils/platform.ts
export const showToast = (message: string) => {
  // #ifdef H5
  showToast(message) // Vant提示
  // #endif

  // #ifdef MP
  uni.showToast({ title: message, icon: 'none' })
  // #endif
}
```

## 开发环境要求

### Node.js

- 推荐版本：**Node.js 18.x 或更高**
- 推荐包管理器：**pnpm 8.x 或更高**

### IDE推荐

- **VS Code** + Vue Language Features (Volar)
- **WebStorm**（商业IDE）

### 浏览器

- Chrome/Edge（开发调试）
- Safari（iOS兼容性测试）

## 项目初始化模板

```bash
# 创建项目
pnpm create vite my-app --template vue-ts

# 安装依赖
cd my-app
pnpm install

# 安装uni-app相关依赖（如需）
pnpm add @dcloudio/uni-app pinia
```

## 版本管理

遵循[语义化版本](https://semver.org/lang/zh-CN/)规范：

- **主版本号**：不兼容的API修改
- **次版本号**：向下兼容的功能性新增
- **修订号**：向下兼容的问题修正
